# CSM v2.1 概念状态机架构

## 概述

CSM (Conceptual State Machine) 是一种层次化认知架构。其核心思想是以**显式状态机结构**替代 Transformer 的隐式注意力模块，通过**场景感知路由**实现稀疏激活，通过**双层 Gated SSM**实现元认知，通过**因果图**实现反事实推理与因果效应估计。

经历了5轮实证迭代，从 CSM v2.0 的 12% 评分 (对 PPO+LSTM baseline 的 43%) 改进到现在的 **69%** 成功率 — 比 baseline 提高 **57%**。在 **P3** 阶段 (元认知生效) 后，成功率开始显著上升。

## 各模块详述

### 1. Encoder (56 -> 64)

两层 MLP + GELU。将环境状态（部分可观测 3x3 视野 one-hot + 2 个内部状态）编码为 64 维向量 `h`。

### 2. C-1 元目标层 (8维)

输入：`h` + 3 个控制信号

输出：`goal` = `tanh(Linear(h + signals))` (8维)

控制信号：

- 好奇心: 基于**学习进度**（预测误差下降率）而非原始预测误差，防止噪声干扰（例如: 地面地毯和宝石的随机关联）
- 能力缺口: 负奖励的短期统计
- 外部奖励: 环境奖励

课程模式：阶段 1-2 时将目标锁为常量（乘以0）。

### 3. 因果图 G

4个离散变量（has_key、door_open、near_treasure、saw_deco）。

检测器：`sigmoid(Linear(h))` → 4 个变量概率

- 用环境地面真值监督检测器（BCE loss）
- 验证：v_key≈0.833, v_door≈0.620

**转移预测**：`predict_delta(var_probs, action_one_hot)`

- 预测变量的变化量（tanh输出）
- 结构方程：`struct_eq[i](parent_signal + action_signal)` -> `delta_v_i` (变更量)
- **为什么用增量**：当 target ≈ input 时，偏置即可最小化损失，结构参数（如 A 矩阵）无梯度。只有预测增量才需要学习 SSM 矩阵。

**稀疏约束**：L0.5（比 L1 更强地压缩弱连接为0）

**反事实**：`counterfactual(var_probs, intervene_idx, intervene_val)`

- do(X=x) 操作：固定干预变量，切断其入边，用新值预测下游

### 4. 场景感知稀疏路由

修复最大的功能点。

**v2.0 原方案：**

```
h_out = h * sparse_mask
        // 随机丢弃85%维度，20步后安全无法进行
        // 学不进就掉，成功率跳水
```

**v2.1 新方案：**

```
enhancement = sum(weights_i * enhancement_vector_i)  // 加权增强向量
alpha = sigmoid(gate(concat(h, goal)))                // 门控强度
h_out = h + alpha * enhancement                        // 加法增强，保证信息无损
```

跳出了乘法掩码的桎梏，让 SSM 的增强信号不断的传递和累积，而不是丢弃。

### 5. S_obj GatedSSM (state_dim=32)

**v2.0 issue**：SSM 推导为 h = A*h + B*x，慢变结构极易形成预测误差接近 0 永远无法更新。梯度爆炸/消失问题。

**v2.1 修复**：采用门控机制，推导为

```
z = sigmoid(W_z * concat(x, S))       // 遗忘门
r = sigmoid(W_r * concat(x, S) + Dr)  // 重置门（受 S_meta 调制）
S_new = (1 - z) * S + z * tanh(W_h * concat(x, r * S))
y = output_proj(S_new)
```

**Dr 由 S_meta 提供**，作为偏置调节重置门，而非直接修改 SSM 矩阵。这让 SSM 既稳定又可调制。

### 6. S_meta 元认知层 (state_dim=16)

核心原则：S_meta **不要处理外部输入**，只理解并干预 **推理过程**。

**v2.0 issue**：输入 h 绝对值，推理过程步间几乎恒定，导致输出 igtp 永远为0.5（不动点）。

**v2.1 修复**：输入为 `delta_S_obj`（状态变化）+ `action_repeat_count`（重复动作）+ 3 个标量信号 → 共 36 维输入。

输出：

1. **Dr偏置** 来调制 S_obj 的思维
2. **4维 interp 信号**：
   - interp[0]: 循环警告 → 重复动作
   - interp[1]: 置信度 → 预测误差低
   - interp[2]: 需要探索 → 预测误差高
   - interp[3]: 需要利用 → 正奖励

**辅助监督**：将 interp 与地面真值 target（循环、误差、奖励）做 MSE 损失项 → 从代码上封住"自指系统仍需知觉锚点"的漏洞。

### 7. 槽位工作记忆 (4 槽位，64维)

解决 SSM 类模型在长序列上精確召回弱的问题。

- 读取：`attn = softmax(slots @ query / sqrt(d))`，加权求和
- 写入：`slots += write_weights * (query - slots) * 0.3` (软门控)
- 初始化为可学习的 **空 token**

### 8. 层次概念状态 + 跳跃连接

```
c2 = Linear(concat(S_obj(32), slot_read(64), h(64)))
```

跳跃连接是 69% 成功率的关键：即使中间模块全失败（如 S_obj、槽位记忆），信息通过跳跃连接直通到输出。这是 ResNet 预见的洞见：**深层模块应该是残差/增强，不是替代**。

### 9. 内部奖励

`R_internal = 0.5 * learning_progress + 0.3 * info_gain`

- **学习进度**：`max(0, 旧误差 - 新误差)` → 只奖励"正在变得可预测"的新颖性（12% → 69% 核心突破）
- **信息增益**：因果变量置信度的平均绝对变化率
- 用增量代替绝对误差，防止噪声干扰（例如: 随机壁的联想）

### 10. 控制符系统

时间节律 / 导航 / 结构 / 生成。在当前简化版本中未显式实现，但已预留接口（面对称触发点）。

## 训练范式

### 课程三阶段训练

| 阶段 | 可用模块 | 冻结模块 |
|------|----------|---------|
| P1 核心（前 800 episodes）| Encoder, SSM, ActionHead, WorldModel, Slot | S_meta干预、目标调制 |
| P2 因果（前 1600 episodes）| +因果检测、因果转移 | S_meta干预 |
| P3 元认知（full） | 全部解锁 | 无 |

### 损失函数

P1:

```
L = L_policy (REINFORCE) + 0.5 * L_value + 
    L_world (SSM 预测误差) + L_causal_detect (BCE) +
    熵地板
```

P2 增加:

```
+ L_causal_predict (MSE, predict_delta) + 0.5 * L_causal_sparse (L0.5)
```

P3 增加:

```
+ 0.2 * L_interp (MSE, 地面真值) + L_internal_reward
```

### 训练细节

- **优化器**：Adam，采用差异化学习率：
  - 核心模块（Encoder/SSM/Action）：3e-4
  - 路由/因果模块：1e-4
  - 元认知模块（S_meta）：5e-5
- **梯度裁剪**：max_norm=0.5 防止炸
- **熵地板**：0.3（低于此值加惩罚，防止策略坍缩）
- **环境种子轮换**：5个环境轮换训练，防过拟合
- **单 episode REINFORCE**：PPO 做批量训练太慢，单 episode + 标准化 + 熵地板在 CPU 上实用。

## 环境评定（测试场地）

**7x7 因果网格世界**：

- 有钥匙 K → 中间门 D → 宝藏 T
- 随机装饰 5 → 干扰变量，用来测试能否区分因果 vs 相关
- **部分可观测**：3x3视野 + 内部状态（has_key, door_open）
- **动作**：6个（up/down/left/right/pickup/use）
- **塑形奖励**：每步靠近当前子目标 +0.1，拿到钥匙 +0.5，开门 +0.5，拿到宝藏 +1
- **地面真值接口**：`get_ground_truth()` 或 `get_gt()`

## 迭代教训（最重要）

| 教训 | 影响 | 解决 |
|------|------|------|
| **裸递归SSM 不可训练** | 原预测误差 → 增加梯度爆炸 | 门控结构（GRU/LSTM/SSM 稳定化） |
| **乘法掩码 = 信息瓶颈** | 丢弃85%信息，下游 **无药可救** | 加法增强 h + enrichment |
| **跳跃连接是生命线** | 没借助，架构复杂度就是负债 | ResNet洞见：**残差 = 状态amba + 增强 + 原始** |
| **预测绝对值无法学结构** | 偏置即可伪正确 | 用增量捕获信息变化 |
| **不变输入导致不动点** | S_obj 全恒定，S_meta 锁在0.5 | 输入 delta_S_obj 显著变化 |
| **自指仍需地面真值锚定** | 无监督 → 一直坚持0.5 | MSE(interp, target) 已生效 |
| **课程解除鸡生蛋循环** | 元认知模块晚开 → 基础模块学不进 | 层次释放解锁 |
| **单episode REINFORCE 高方差但可靠** | 无法纯 PPO | 标准化 + 熵地板 |
| **地面真值是桥梁** | 纯自监督难以收敛 | 检测器先学 → 然后从增量预测 |
| **好奇心 = 最大价值实现** | 噪声影响严重 | 用学习进度捕捉有用新颖性 |

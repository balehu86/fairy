C₋₁ 元目标层
新增模块 维度 d = 16 位于 C₀ 之上
设计动机

原有架构是纯响应式的——等待输入、处理、输出，没有自驱动力。C₋₁ 层的目的是让系统能产生自己的"想做什么"，并将其向下传递给场景路由器，将注意力从被动响应变为目标主动引导。

这是从"工具"到"有限自主系统"最关键的结构性改变。

三个子组件
好奇心信号
ICM 模块
预测误差大的区域 → 驱动探索。误差高 = 有趣 = 值得关注。
能力缺口检测
历史误差分析
任务失败模式分析 → 识别系统不擅长的领域 → 驱动针对性学习。
目标层次树
长期→子目标分解
长期目标动态分解为短期子目标，支持多步规划。
目标冲突仲裁
软约束满足
多目标之间的优先级动态调整，避免目标之间相互干扰。
关键机制：反向调制场景路由
原有（输入驱动）

路由器只根据当前输入决定激活哪些维度。
"我看到了什么" → 激活相关维度

改进后（目标驱动）

C₋₁ 的目标向量对路由权重施加偏置。
"我想做什么" → 也影响激活哪些维度

边界说明

C₋₁ 的目标由内部奖励驱动，但内部奖励本身（好奇心、一致性）仍是人工设计的。系统自己产生的"目标"，本质上是对人工定义奖励函数的优化。真正的"想要"从哪里来——这是哲学问题，不是工程问题，目前没有答案。

⟳
S_meta 元认知层
新增模块 与 S_obj 并行运行 每 N 步更新一次
核心思想

原架构中"观察自身状态"只是读取状态向量——这不是真正的元认知。真正的自指需要系统能对自己的推理过程建模，而不只是读取当前激活值。

S_meta 的输入不是外部数据，而是 S_obj 的推理轨迹本身。它的输出是 ΔA——直接修改 S_obj 的 A 矩阵参数。这是自我修改的最小可行实现。

输入 vs 输出
S_meta 的输入

· S_obj 的推理轨迹
· 历史预测误差
· 控制符使用统计
· 内部奖励信号

S_meta 的输出

· ΔA（修改 A 矩阵）
· 内生控制符
· 置信度评估
· 目标层更新信号

状态更新方程
// 对象层（每步更新） S_obj' = (A_base + ΔA) · S_obj + B · x + ControlNet(c) // 元认知层（每N步更新） S_meta' = f_meta(S_meta, trajectory(S_obj), reward) // 元认知层输出 ΔA = MLP_A(S_meta) // 修改对象层动力学 ctrl = MLP_ctrl(S_meta) // 内生控制符 conf = sigmoid(MLP_c(S_meta)) // 置信度
与普通自我观察的根本区别
仅读取状态（原有）

把当前状态向量作为额外输入通道读取。没有修改推理逻辑的能力，只是"看到"自己。

修改动力学（改进后）

S_meta 可以改变 A 矩阵的部分参数，即修改 S_obj 的状态转移规则。系统不只"看到"自己，还能改变自己的思维方式。

⇢
因果图层 G
新增模块 稀疏矩阵 与状态空间协同演化
为什么需要因果推断

当前架构（包括所有 LLM）本质上是统计关联学习，无法区分相关和因果。经典反例：夏天冰激凌销量↑ 和溺水率↑ 高度相关，但买冰激凌不会导致溺水。

Pearl 因果层级：关联（观察到 A→B）→ 干预（do(A)→B）→ 反事实（如果 A 没发生，B 会如何）。当前 LLM 基本停留在第一层，本模块目标是覆盖第二、三层。

普通更新 vs 因果更新
// 普通状态更新（统计关联） S' = f(S, x) // 观察到 x 后状态如何变化——只能学习关联 // 因果状态更新（干预表征） S' = f(S, do(x)) // 如果强制 x 发生，状态预测如何变化——区分相关 vs 因果
因果图 G 的工作机制
学习方式
反事实训练数据
通过随机干预实验和结构方程建模从数据中学习因果关系。
实时更新
每步更新边权重
每次状态更新后同步更新 G 的因果置信度，稀疏约束防止假阳性。
推理时用途
反事实查询
回答"如果我做 X，Y 会发生吗"——干预效果预测与归因分析。
实现难度
极高
需要大量反事实训练数据，是架构中最具挑战性的部分。
为什么这是"鹦鹉"到"理解"最关键的一步

没有因果推断，系统只能记住"A 和 B 经常一起出现"。有了因果推断，系统可以回答"如果 A 不存在，B 还会发生吗"。这是真正理解世界的基础，也是规划和干预能力的前提。

◎
内部奖励计算器
新增模块 驱动 C₋₁ 元目标层 不依赖外部标注
三个奖励分量
r₁ 好奇心奖励
r₁ = |预测误差|
高预测误差 = 遇到新事物 = 有探索价值。驱动系统主动寻找不确定区域。
r₂ 任务完成奖励
r₂ = 目标达成度
稀疏奖励 + 成型（shaping）。衡量 C₋₁ 产生的目标是否被实现。
r₃ 一致性奖励
r₃ = 状态一致性
惩罚自相矛盾的内部状态，鼓励连贯、一致的世界模型。
总奖励公式
R = α·r₁ + β·r₂ + γ·r₃
权重 α, β, γ 可学习，在阶段E中自动调整以平衡探索与利用。
信号流向
C₃ 概念状态 ↓ 内部奖励计算器 → R = α·r₁ + β·r₂ + γ·r₃ ↓ C₋₁ 元目标层（更新目标向量） ↓ 场景路由器（偏置激活权重） ← 目标驱动注意力
局限性

内部奖励的定义本身（好奇心、任务完成、一致性）仍然是人工设计的。这意味着系统的"内在动机"终究是对人工奖励函数的优化，而非真正自主涌现的欲望。这个边界目前无法突破。

◻
多模态嵌入层
原有模块 编码阶段执行一次
功能

将不同模态的原始输入映射到同一状态空间。训练目标是使配对的多模态输入在状态空间中占据相近位置（对比学习 + 循环一致性）。

E_vision: image → ℝ^D E_audio: audio → ℝ^D E_text: text → ℝ^D E_action: action → ℝ^D
v2 的变化

嵌入层本身无改动。但嵌入结果现在同时进入 C₋₁（目标层）和因果图层 G，以便 G 能从输入模式中识别潜在因果结构。

◈
场景感知稀疏激活
核心改进 新增目标驱动路由
原有机制（保留）

64个可学习场景原型 · Gumbel-Softmax 可微选择 · 温度退火 5.0→0.5 · 激活 40–55% 维度。

改进：C₋₁ 反向调制
原有路由逻辑

路由权重 = f(当前输入)
"我看到什么" → 激活哪些维度

改进后路由逻辑

路由权重 = f(当前输入) + g(C₋₁目标向量)
"我想做什么"也参与激活决策

这个改动使注意力从被动响应变为可被目标主动引导——系统不只响应"发生了什么"，也响应"我想要什么"。

≋
S_obj 对象层 Mamba
核心改进 A 矩阵受 S_meta 调制
改进后的状态更新方程
// 原有方程 S' = A · S + B · x + ControlNet(c) // 改进后 S' = (A_base + ΔA) · S + B · x + ControlNet(c) y = C · S' // 其中 ΔA 由 S_meta 每 N 步提供一次 // A_base 是可学习的基础转移矩阵 // ΔA 是元认知层对动力学的实时调整
ΔA 的含义

A 矩阵决定系统"如何思考"——哪些信息长期保留、哪些快速衰减。S_meta 通过修改 ΔA，实质上改变了 S_obj 的记忆机制和思维习惯。这是架构中唯一真正的自我修改路径。

设计哲学

状态空间已包含所有必要信息。输出只是换基底——从状态基底变换到词表/像素/动作基底。没有独立的"语言模型"，语言能力是状态空间的涌现属性。

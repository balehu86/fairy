设计铁律 — 9轮实证换来的禁忌
每条背后都是一次失败 违反即崩溃
架构层铁律
裸递归 S' = A·S + B·x → 特征值>1爆炸, <1消失。必须用门控结构 (1-z)·S + z·h_hat
乘法掩码 h *= mask → 丢弃85%信息不可恢复。必须用加法增强 h + α·enhancement (初始化退化为恒等映射)
模块串联无跳跃 → 串联瓶颈, 上游失败全部下游无信号。必须有跳跃连接让信息直通输出
S_meta输入S_obj绝对值 → 步间不变导致SSM不动点。必须输入ΔS_obj (状态变化量)
interp走SSM路径 → 恒定输入收敛到不动点, 丧失即时响应。interp必须走前馈路径
interp多输出共享网络 → 梯度量级大的目标绑架量级小的(置信度绑架循环信号)。循环信号必须用独立输出头
训练层铁律
每步训练因果转移 → 非事件步输出0→鼓励模型输出0, 事件步正梯度被淹没。必须只在事件步训练且5-10x加权
sparsity同时作用于action_effects → 每步50次推力vs事件步3次正梯度, action_effects被推向0。sparsity仅对var_adj
因果检测器无地面真值监督 → 纯自监督不知道该检测什么, 不收敛。必须BCE(var_probs, ground_truth)
var_adj用间接梯度(predict_delta反传) → 所有边接受相同方向梯度→均匀0.47无法分化。必须用共现监督给每条边独立梯度+先验初始化
action_effects走sigmoid → sigmoid(0)=0.5是梯度甜蜜点, 平衡在0.47。必须直接参数化从0开始, 只有正梯度推离0
全模块同时训练 → 鸡生蛋: S_meta干扰S_obj, 因果图需检测器先就绪。必须课程训练
好奇心用原始预测误差 → 电视效应: 随机噪声吸引注意力。必须用学习进度 max(0, 旧误差-新误差)
奖励只有稀疏终点 → 94%episode无信号, 学不到因果链。必须加塑形奖励(靠近子目标+里程碑)
因果推理铁律
预测绝对值 predict(var_probs) → 偏置≈target即可解, 无需学习转移。必须预测增量 delta
var_adj无先验初始化 → 所有边从同一起点出发, 共现监督使所有相关边同时升高无法分化。必须先验初始化打破对称
var_adj只用共现监督 → 共现无法区分因果方向(i→j vs j→i)。必须先验+共现组合, 或加入时序偏置
var_adj全0.45时判断因果效应 → 均匀传播的假阳性, 任何干预等量影响所有变量。因果效应>0.6且var_adj分化才是真因果

S_obj GatedSSM 核心
实证修正 替代裸递归 Mamba state_dim=32
原始方案 vs 修正方案
原始: 裸递归 Mamba

S' = A·S + B·x + ControlNet(c)
A矩阵决定记忆衰减, 特征值>1→梯度爆炸, <1→消失。
理论优雅, 实证崩溃: Loss震荡 -0.3到1.5。

修正: GatedSSM

z = σ(W_z · [x, S]) // 遗忘门
r = σ(W_r · [x, S] + Δr) // 重置门(受ΔA调制)
S' = (1-z)·S + z·tanh(W_h · [x, r·S])
稳定性保证: (1-z)∈[0,1] 约束特征值。

完整状态更新方程
// 输入: x(64维), S_obj(32维) // ΔA: 来自 S_meta 的低秩调制信号 combined = concat(x, S_obj) // 96维 z = sigmoid(W_z(combined)) // 遗忘门: 控制保留多少旧状态 r = sigmoid(W_r(combined) + delta_A_bias) // 重置门: 受低秩ΔA调制 h_hat = tanh(W_h(concat(x, r * S_obj))) // 候选新状态 S_obj' = (1 - z) * S_obj + z * h_hat // 稳定更新 y = output_proj(S_obj') // 32→64
ΔA 的两种调制方式
当前实现: 重置门偏置

delta_signal → r偏置 × 0.1
安全但影响弱: 只控制"忘多少", 不改变动力学基本结构
P3解锁后成功率无明显额外提升

优化方案: 低秩ΔA

ΔA = W_up(W_down(S_meta)) · σ(norm_gate)
秩=4, 谱范数可控, 既能修改动力学又防漂移
详见"低秩ΔA机制"面板

第1轮: 裸递归Loss震荡-0.3→1.5, 成功率12%。门控替代后Loss稳定收敛。门控的(1-z)∈[0,1]是稳定性保证——无论W怎么初始化, 特征值不会爆炸。

跳跃连接系统
实证修正 最大单项改进 ResNet洞见
设计原理

原始架构是严格分层串联: 输入→嵌入→概念层→输出。任何一层失败, 下游全部无信号。这是串联瓶颈——系统可靠性等于各层可靠性之积。

ResNet的洞见: 深层模块应该是残差/增强, 不是替代。即使SSM、槽位、场景路由全失败, 信息通过h直通到ActionHead。高层模块在"不拖后腿"和"独立贡献"之间, 跳跃连接确保了至少"不拖后腿"。

跳跃连接点
h (64维) ─────────────────────────────────┐ │ │ ├──→ 场景路由(h+α·enh) │ │ │ │ │ ├──→ GatedSSM(S_obj) │ │ │ │ │ │ │ ├──→ c2 ─────────────────┤ │ │ │ │ │ ├──→ SlotMemory(slot_read) ───────┤ │ │ │ └──→ c2 = Linear(S_obj, slot_read, h) ←──┘ 32 + 64 + 64 = 160 → 64 关键: h 必须在 c2 的输入中! 移除后成功率暴跌。
层次概念空间的层间跳跃

原始C₀→C₁→C₂→C₃严格串联。优化后每层都有到输出的独立跳跃路径, 任何层级的抽象都能直接参与决策:

C₀ (64维) ──→ output_proj C₁ (256维) ──→ output_proj // 每层独立跳跃 C₂ (512维) ──→ output_proj // 不依赖上层就能贡献 C₃ (1024维)──→ output_proj
第4轮: 加跳跃连接后 69%→79%(最终)。消融: 移除h跳跃后成功率暴跌。跳跃连接做了大部分重活, 高级模块从"拖后腿"向"独立贡献"过渡。

低秩 ΔA 机制
优化升级 替代全秩ΔA LoRA-style
演进路径
原始构想: 全秩 ΔA

S' = (A + ΔA)·S + B·x
ΔA: 32×32 = 1024参数, 可完全改变A矩阵
危险: 微小扰动可导致特征值>1, 系统动力学崩溃
未实证验证

当前实现: 重置门偏置

delta_signal → r偏置 × 0.1
安全但弱: 只控制"忘多少", 不改变动力学
量化结果: P3解锁后无显著额外提升
调制幅度可能太小

优化方案: 低秩 ΔA (LoRA-style)

ΔA = W_up(W_down(S_meta)) · σ(norm_gate)
W_down: 16→4 (压缩), W_up: 4→32 (展开)
秩=4: 只影响4个主方向, 谱范数可控
σ(norm_gate): 自适应幅度控制, 防漂移
核心优势: 既能修改动力学(超越重置门偏置), 又通过低秩约束防止失控(超越全秩ΔA)

实现代码
class LowRankDeltaA(nn.Module): def __init__(self, meta_dim=16, state_dim=32, rank=4): super().__init__() self.down = nn.Linear(meta_dim, rank) # 16→4 self.up = nn.Linear(rank, state_dim) # 4→32 self.norm_gate = nn.Linear(meta_dim, 1) # 幅度控制 # 初始化: up.weight 接近0, ΔA初始≈0 nn.init.normal_(self.up.weight, 0, 0.01) nn.init.zeros_(self.up.bias) def forward(self, S_meta): delta = self.up(self.down(S_meta)) # 32维 scale = torch.sigmoid(self.norm_gate(S_meta)) # [0,1] return delta * scale # 自适应幅度 # 在 GatedSSM 中使用: # r = sigmoid(W_r(combined) + delta_A) # 重置门调制 # S' = (1-z)*S + z*tanh(W_h @ [x, r*(A_base+ΔA)@S]) # 动力学调制
为什么低秩足够

元认知不需要修改S_obj的每一个自由度。S_meta观察到"推理陷入循环"或"预测误差偏高"时, 需要调整的是少数几个动力学主方向——增大遗忘率、改变记忆保持策略。这些是低维操作, 秩4足以表达。

类比: LoRA在LLM微调中秩4-8就足以改变行为, 因为"任务适应"是低维操作。元认知对动力学的修改同样是低维操作。

全秩ΔA: 32×32参数可完全改变A矩阵, 微小扰动导致特征值漂移→动力学崩溃。低秩约束+幅度控制是安全使用ΔA的必要条件。

为什么前馈不走SSM

SSM的动力学会让恒定输入的输出收敛到不动点。如果action_repeat连续20步为高值, SSM会适应这个"常态"并输出接近0——系统不再认为循环是异常。前馈路径保证输入变化→输出立即变化, 不受历史状态影响。

第7-9轮: interp走SSM→不动点锁死0.5。输入含ΔS_obj→32/36维噪声淹没有效信号。共享网络→梯度绑架, 循环信号0.086→0.039方向反转。独立输出头是唯一验证过的解决方案。
interp走SSM → 不动点。interp输入含ΔS_obj → 有效信号被零值淹没。多输出共享网络 → 梯度绑架。interp监督target二值化 → 训练信号太弱。
